# FAQ

## General

**Q: What is ab1tools?**
A: ab1tools is a comprehensive command-line toolkit for AB1 Sanger chromatogram files. It can generate, visualize, analyze, convert, compare, trim, and extract AB1 files — filling the "samtools for AB1" gap in bioinformatics.

**Q: Does ab1tools replace Sanger sequencing?**
A: No. ab1tools generates *Sanger-like* chromatograms from NGS data, bridging the two technologies. It enables researchers to visualize NGS results using familiar chromatogram tools (SnapGene, Chromas) and provides a "virtual Sanger" for variant review.

**Q: Is ab1tools limited to Nanopore data?**
A: No. While primarily developed for Nanopore amplicon sequencing, ab1tools works with **any sequencing platform** that produces standard BAM files (Illumina, PacBio, Ion Torrent, MGI, etc.). The core algorithm uses standard BAM pileup — no platform-specific code.

**Q: Can ab1tools read real Sanger AB1 files from ABI sequencers?**
A: Yes. `ab1tools view`, `convert`, `stats`, `call`, `compare`, `trim`, `extract`, and `plot-ab1` all work with real Sanger AB1 files. The `from-seq` and `single` modes generate new AB1 files.

## Variant Detection

**Q: What is SDVC?**
A: Signal Deconvolution-based Variant Calling — ab1tools' algorithm for detecting variants directly from 4-channel chromatogram trace data. It features dynamic thresholding, Bayesian prior support, and multi-position consistency.

**Q: What is the limit of detection?**
A: In control-free mode (Mode 1): ~2-3% minor allele frequency. In control-enhanced mode (Mode 2, with matched control): <1%. See the LOD analysis in `docs/research/phase2_variant_caller/results/lod_analysis.md`.

**Q: How does SDVC compare to TIDE?**
A: TIDE requires a mandatory control trace and uses NNLS decomposition. SDVC works both with and without a control (dual-mode) and uses signal-level peak deconvolution with adaptive thresholding. SDVC's control-enhanced mode achieves comparable or better sensitivity.

**Q: Can I use SDVC on real Sanger AB1 files (not generated by ab1tools)?**
A: Yes. `ab1tools call` works on any valid AB1 file, whether from a real ABI sequencer or generated by ab1tools.

## Format and Compatibility

**Q: Which viewers can open ab1tools-generated AB1 files?**
A: SnapGene, Chromas, FinchTV, ApE, 4Peaks, Biopython, and any tool that reads ABIF format. We have verified compatibility with SnapGene, Chromas, FinchTV, and ApE.

**Q: What is the maximum sequence length?**
A: ~6,553 bases at default spacing (10). For longer sequences, ab1tools automatically reduces spacing. AB1 peak positions are uint16 (max 65,535 trace points).

**Q: Can I convert AB1 back to FASTA/FASTQ?**
A: Yes. `ab1tools convert sample.ab1 -f fasta` or `-f fastq`. Also supports JSON and CSV output.

## Troubleshooting

**Q: I get "BAM file has no references" error.**
A: The BAM file needs to have reads aligned to a reference. Make sure it was produced by an aligner (minimap2, bwa, etc.) and is sorted and indexed (`samtools sort`, `samtools index`).

**Q: PNG files are very large.**
A: Use `--no-plot` to skip PNG generation. Or use `--dpi 100` (default 150) for smaller files. For specific regions only, use `plot-ab1 --start --end`.

**Q: Quality scores are all the same in from-seq mode.**
A: In `from-seq` mode, each position has 100% frequency for its base, giving uniform Q60 quality. This is correct — there's no ambiguity. Use `single` mode with BAM data for position-specific quality variation.
