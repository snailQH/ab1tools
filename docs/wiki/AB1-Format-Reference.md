# AB1 Format Reference

Technical reference for the ABIF (Applied Biosystems Instrument Format) binary format as implemented in ab1tools.

---

## Format Overview

The AB1 file format (extension `.ab1`) was developed by Applied Biosystems (now Thermo Fisher Scientific) for storing Sanger sequencing chromatogram data. It uses the ABIF container format, which consists of a fixed header followed by tagged data blocks.

### File Structure

```
Offset    Size     Content
───────────────────────────────
0-3       4 bytes  Magic: "ABIF" (0x41424946)
4-5       2 bytes  Version: 101 (big-endian uint16)
6-33      28 bytes Root directory entry ("tdir1")
34-127    94 bytes Padding (zeros)
128-...   variable Data blocks (4-byte aligned)
...-EOF   variable Directory entries (28 bytes each)
```

### Directory Entry Structure (28 bytes)

```
Offset  Size  Field         Type       Description
──────────────────────────────────────────────────
0       4     tag_name      ASCII      e.g., "DATA", "PBAS", "PLOC"
4       4     tag_number    uint32 BE  e.g., 9, 10, 11, 12
8       2     element_type  uint16 BE  Data type code
10      2     element_size  uint16 BE  Bytes per element
12      4     num_elements  uint32 BE  Number of elements
16      4     data_size     uint32 BE  Total data bytes
20      4     data_offset   uint32 BE  File offset (or inline if ≤4 bytes)
24      4     dataHandle    uint32     Unused (zeros)
```

### Element Type Codes

| Code | Name | Size | Description |
|------|------|------|-------------|
| 1 | BYTE | 1 | Unsigned 8-bit integer |
| 2 | CHAR | 1 | ASCII character |
| 4 | SHORT | 2 | Signed 16-bit integer (big-endian) |
| 5 | LONG | 4 | Signed 32-bit integer (big-endian) |
| 18 | PSTRING | var | Pascal string (length byte + data) |
| 19 | CSTRING | var | C string (null-terminated) |

## Tags Written by ab1tools

| Tag | Type | Content | Description |
|-----|------|---------|-------------|
| DATA9 | SHORT[] | int16 BE array | G channel trace data |
| DATA10 | SHORT[] | int16 BE array | A channel trace data |
| DATA11 | SHORT[] | int16 BE array | T channel trace data |
| DATA12 | SHORT[] | int16 BE array | C channel trace data |
| FWO_1 | CHAR[4] | "GATC" | Channel order |
| LANE1 | LONG | 1 | Lane number |
| PBAS1 | CHAR[] | ASCII string | Base calls |
| PCON2 | BYTE[] | uint8 array | Quality scores (0-60) |
| PDMF1 | PSTRING | "kb.mob" | Mobility file name |
| PDMF2 | PSTRING | "kb.mob" | Mobility file name |
| PLOC1 | SHORT[] | uint16 BE array | Peak positions |
| S/N%1 | SHORT[4] | int16 BE | Signal-to-noise ratios (G, A, T, C) |
| SMPL1 | PSTRING | sample name | Sample name |
| CMNT1 | PSTRING | comment | "Generated by ab1tools" |

### Channel Order: GATC

The ABIF format stores trace channels in **GATC** order (not ACGT):

- DATA9 = **G** (Guanine)
- DATA10 = **A** (Adenine)
- DATA11 = **T** (Thymine)
- DATA12 = **C** (Cytosine)

The FWO_1 tag ("GATC") tells viewers which channel corresponds to which base.

### Peak Positions

Peak positions (PLOC1) are stored as **big-endian uint16**, giving a maximum value of 65,535. For a trace with spacing=10, this limits sequences to ~6,553 bases. ab1tools automatically reduces spacing for longer sequences.

### Quality Scores

Quality scores (PCON2) follow the phred convention:
```
Q = -10 × log₁₀(P_error)
```

Where `P_error = 1 - f_max` (f_max = frequency of the called base). Scores are capped at 60 (Q60 = 1 in 1,000,000 error rate).

## Reading AB1 Files Programmatically

### With ab1tools

```python
from ab1tools.abif_reader import read_ab1

traces, base_calls, peak_positions, sample_name = read_ab1("sample.ab1")
# traces: {"A": np.array, "C": np.array, "G": np.array, "T": np.array}
# base_calls: "ACGTACGT..."
# peak_positions: [0, 10, 20, 30, ...]
```

### With Biopython

```python
from Bio import SeqIO
record = SeqIO.read("sample.ab1", "abi")
print(record.seq)
data = record.annotations["abif_raw"]
# data["DATA9"] = G channel, data["DATA10"] = A channel, etc.
```
